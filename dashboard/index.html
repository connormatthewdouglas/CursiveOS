How to read these numbers (honestly)

  • Cold-start = how fast the machine produces the first AI token after idle. Lower is better; this is the project's most reliable measurement.
  • Network figures come from a lab simulation (loopback), not a real internet link. On real hardware the gain is essentially "switch TCP to BBR" — real under packet loss, but not a CursiveOS invention. Treat the big percentages as lab-only.
  • Idle power is the machine's package power at rest (watts). Lower is better.
  • A result on one machine is not a fleet result — gains can be hardware-specific (e.g. the cold-start win is large on the Arc desktop, ~zero on the laptop).
